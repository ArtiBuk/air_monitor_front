# air_monitor_front

Frontend для `air_monitor_back`: интерфейс управления пайплайном мониторинга и краткосрочного прогнозирования качества воздуха.

Приложение закрывает рабочий сценарий целиком: авторизация, сбор наблюдений, сборка датасетов, обучение моделей, генерация прогнозов, ретропроверка, оценка по факту, запуск исследовательских экспериментов и контроль фоновых задач.

Интерфейс ориентирован на предметную область Норильского промышленного района и делает акцент на `AQI` как на итоговом индикаторе качества воздуха.

## Возможности

- вход и регистрация поверх cookie-based auth backend;
- обзорная панель по состоянию пайплайна;
- скачивание итогового PDF-отчёта прямо с обзорной панели;
- выделенный блок `AQI` на обзоре с последней точкой прогноза и категорией риска;
- интерактивная карта воздуха с временной шкалой, слоями, hot spots и городским фоном;
- сбор наблюдений и просмотр последних записей;
- русскоязычные названия метрик вместо сырьевых backend-id;
- семантическая подсветка показателей: `Норма`, `Выше нормы`, `Критический`;
- сборка `DatasetSnapshot`;
- обучение и просмотр версий моделей;
- генерация прогнозов и `backtest`;
- оценка прогноза по фактическим данным;
- запуск экспериментальных прогонов и серий;
- мониторинг фоновых и отложенных задач;
- скачиваемая инструкция по интерфейсу прямо из шапки приложения.

## Экранная модель

- `/login` — вход и регистрация.
- `/app` — обзор пайплайна.
- `/app/air-map` — карта качества воздуха по постам и городскому фону.
- `/app/observations` — сбор наблюдений и фильтры по последним данным.
- `/app/datasets` — сборка и просмотр срезов датасета.
- `/app/models` — обучение, активная модель и лидерборд.
- `/app/forecasts` — прогнозы, ретропроверка и оценка.
- `/app/experiments` — исследовательские прогоны и серии.
- `/app/tasks` — отслеживание фоновых и отложенных задач.

## Что важно в поведении интерфейса

- Фронт старается автоподставлять параметры последнего датасета, активной модели и последнего прогноза.
- В выпадающем списке прогнозов показываются только модели со статусом `ready`.
- Кнопка оценки прогноза по факту активируется только после появления фактических наблюдений на весь горизонт.
- Отслеживаемые фоновые задачи хранятся в `localStorage`, чтобы не теряться при переходах между страницами.
- Любая фоновая операция может быть добавлена в локальный мониторинг вручную по `task_id`.
- На странице `Обзор` счетчики берутся из отдельного overview endpoint и показывают реальные totals, а не длину урезанных списков.
- Блок автосбора на `Обзоре` показывает фактическую конфигурацию hourly-задачи backend.
- С overview-страницы можно скачать итоговый PDF-отчёт, который backend формирует из реальных данных системы.
- Страница карты использует `OpenFreeMap + MapLibre`, новый backend snapshot `/api/monitoring/air-map` и временную шкалу `4 дня назад + 1 день вперёд`.

## Требования

- Node.js 20+
- npm 10+
- запущенный backend `air_monitor_back`

Для разработки предполагается, что backend доступен по адресу:

```text
http://127.0.0.1:8000
```

## Конфигурация

Основные переменные окружения:

- `VITE_BACKEND_TARGET` — адрес backend для Vite proxy в dev-режиме.
- `VITE_API_BASE_URL` — базовый URL API для production-сборки.
- `VITE_AIR_MAP_STYLE_URL` — URL style для страницы карты воздуха, по умолчанию используется `OpenFreeMap Liberty`.

Быстрый старт:

```bash
cp .env.example .env
```

## Docker deploy

Frontend deploy-контур собирает статический билд и отдаёт его через `nginx`, одновременно проксируя API-запросы на backend внутри общей Docker-сети.

Основные команды:

```bash
make up
make build
make ps
make logs
make down
```

Если менялся код интерфейса, одного `make up` недостаточно: сначала нужен `make build` или `make rebuild`, потом уже `make up`.

Если backend внутри сети доступен под другим именем или портом:

```bash
make up BACKEND_UPSTREAM_HOST=custom-backend BACKEND_UPSTREAM_PORT=8000 FRONTEND_PORT=8080
```

Если нужно поднять оба репозитория одной командой, запуск делается из backend:

```bash
cd /path/to/air_monitor_back
make deploy-up-all FRONTEND_DIR=/path/to/air_monitor_front
```

## Пользовательская инструкция

В шапке приложения рядом с блоком пользователя есть кнопка `Инструкция`. Она скачивает полный markdown-гайд по интерфейсу с описанием разделов, полей форм и рабочих сценариев.

Исходный файл лежит в:

```text
public/air-monitor-user-guide.md
```

Гайд стоит обновлять одновременно с изменением экранов, потому что он скачивается пользователем прямо из production-интерфейса.

## Зависимости и безопасность

- Фронт собран на `Vite 6.4.2+`, потому что `6.4.1` помечался уязвимым в ряде advisory.
- Если IDE продолжает показывать старое предупреждение после обновления зависимостей, нужно перечитать `package-lock.json` и переустановить `node_modules`.

## Связанный backend

Проект работает вместе с backend-репозиторием `air_monitor_back`.

## Статус

Это прикладной интерфейс под конкретный backend и исследовательский пайплайн. Проект не пытается быть универсальным dashboard-конструктором и сознательно повторяет предметные сущности backend почти без абстракций.
